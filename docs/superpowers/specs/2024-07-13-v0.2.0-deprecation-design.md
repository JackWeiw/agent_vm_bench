# v0.2.0 Legacy Scripts Deprecation Design

> **Date**: 2024-07-13
> **Version Target**: 0.2.0
> **Status**: Approved

## Summary

Deprecate legacy scripts (`vm_bench_lite.py`, `create_server.py`) with DeprecationWarning, keeping existing `qemu_monitor` deprecation unchanged. Update version from `0.1.0-alpha` to `0.2.0`.

## Version Update

Update from `0.1.0-alpha` to `0.2.0`. Future version planning will be determined separately.

## Files to Deprecate

| File | Current Status | Action |
|------|---------------|--------|
| `vm_bench_lite.py` | Not deprecated | Add DeprecationWarning in `main()` |
| `create_server.py` | Not deprecated | Add DeprecationWarning in `main()` |
| `qemu_monitor.py` | Already deprecated | No changes |
| `qemu_monitor/` | Already deprecated | No changes |

## Migration Path

Users should migrate to new package-based commands:

| Legacy | New |
|--------|-----|
| `python vm_bench_lite.py -n 100 --browser-mode -wp` | `python -m vm_bench --warmup-only -n 100` |
| `python vm_bench_lite.py -n 100 --browser-mode -bsp 0.5 -t 180` | `python -m vm_bench -bp 0.5 -t 180` |
| `python create_server.py --start_ip ... --n 10` | `python -m vm_bench --create-only -n 10` |

## No Breaking Changes

- All deprecated scripts remain functional
- Import paths unchanged (`from qemu_monitor import X` still works)
- CLI commands unchanged (`python vm_bench_lite.py ...` still works)
- Users receive runtime warnings pointing to migration path
